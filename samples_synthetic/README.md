# 合成樣本說明

本資料夾所有檔案均為**人工合成的示範資料**，不對應任何真實專案、公司或廠商。
用途是讓你能獨立重跑完整流程，親自核對輸出，而不必只採信文字描述。

## 報價單樣本（export_cleaned_quotes.py）

| 檔案 | 展示情境 |
|---|---|
| 範例廠商A報價.xlsx | 標準格式：單列表頭、含小計列（應被排除） |
| 範例廠商B報價.xlsx | 表頭前有標題／說明列、無廠商欄（應靠檔名遞補廠商名） |
| 範例廠商C報價.xls | 舊版 .xls＋內含控制字元（U+0002，模擬複製貼上殘留）＋一筆刻意的複價不符 |
| 附註說明.xlsx | 非報價單內容，表頭信心不足，應被略過且不影響 exit code |

```bash
python export_cleaned_quotes.py --folder ./samples_synthetic/quotes --outdir out_quote
```

預期結果：
- 4 個檔案中，3 個成功、1 個略過（附註說明）
- 品項總數 8（A:3＋B:3＋C:2）；複價驗算「不符」1 筆（範例廠商C 的電線）
- `structured_quote_result.xlsx` 中控制字元已清除；`clean_quote_items.csv`
  同一筆仍保留原始控制字元
- exit code = 0（略過不是失敗）

## 標單樣本（export_cleaned.py）

**範例標單.xlsx** 在 14 列裡濃縮了標單清洗的全部核心情境：

- 雙語重複表頭（應跳過）
- `壹`／`貳` 大分類、`A` 小組名、`1` 次小組名的階層繼承
- 前置情境延伸列（`含基礎型鋼及固定五金`→ 併入後續品項語意）
- 後置規格備註列（`- 含耐壓測試報告` → 掛回上一筆品項）
- **雙數量欄**：`配電盤本體` 數量=2、工地數量=3、複價=3×單價 → 應取 3、
  「另一數量欄」記 2；`PVC管 2吋` 同理應取 100
- 小計列（應排除）
- 跨分類切換（`貳` 之後的品項不受前一分類的情境污染）

```bash
python export_cleaned.py samples_synthetic/boq/範例標單.xlsx --outdir out_boq
```

預期結果：3 筆品項、複價驗算全 OK、`row_audit.csv` 可逐列核對每一列的分類。

## 為什麼分成 quotes/ 與 boq/ 兩個資料夾

標單的欄位（品名／單位／數量／單價）剛好也滿足報價單工具的必要欄位，混放同一
資料夾時標單會被報價單工具當成報價單處理——欄位對得上，但階層語意全部遺失。
這正是主 README「兩種檔案請勿混丟給同一支工具」的具體示範：工具刻意不猜測
檔案類型，資料夾分流是使用者的責任。
